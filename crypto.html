I · Hash Functions

SHA-256 Cryptographic Hash

A deterministic function mapping arbitrary-length input to a fixed 256-bit digest. Any change to the input produces an entirely different output.

Definition
H : {0, 1}* → {0, 1}256
SHA-256(m) = h ∈ {0, 1}256, |h| = 256 bits = 64 hex characters
m Input Data
H(m) SHA-256 Digest
Connection to Thermoeconomics

Preimage resistance is the computational one-way-ness that makes proof-of-work possible. Given h, finding any m such that H(m) = h requires brute-force search over 2256 ≈ 1.16 × 1077 possibilities. Each trial dissipates energy irreversibly — the thermodynamic anchor binding hash computation to physical work.

Block Structure & Proof-of-Work

A block bundles data with a nonce. Mining searches for a nonce satisfying a difficulty target — the core of proof-of-work consensus.

Definition — Block Hash
B = (nblock, η, d)
H(B) = SHA-256(nblock ∥ η ∥ d)
Definition — Mining
Find η ∈ ℕ such that H(B) < T
where T = 2256 − κ, κ = difficulty (leading zero hex digits)
n Block #
η Nonce
d Data
H(B) Block Hash
Target κ = 4 hex zeros
Does not satisfy target
Connection to Thermoeconomics

Proof-of-work is a thermodynamic process. Each nonce trial dissipates energy as heat per Landauer's principle (E ≥ kT ln 2 per bit erasure). The cumulative work is physically embedded in the chain — thermodynamic proof of expended energy.

II · Chain Structure

Blockchain

Blocks are linked by including the previous block's hash. Changing any block invalidates all subsequent blocks, making the history tamper-evident. Scroll right to see the full chain.

Definition — Chain Integrity
Bi = (i, ηi, di, H(Bi−1))
H(Bi) = SHA-256(i ∥ ηi ∥ di ∥ H(Bi−1))
Connection to Thermoeconomics

Hash chaining creates a thermodynamic arrow of time. Each block's hash depends on all preceding computation. Altering a historical block requires re-mining every subsequent block — an exponentially growing energy cost that makes the chain's history practically irreversible, mirroring entropy's irreversibility in physics.

II · Chain Structure

Distributed Blockchain

Multiple peers each maintain their own copy of the chain. Consensus emerges because honest peers accept the longest valid chain. Try tampering with one peer's data — the other copies reveal the forgery.

Definition — Consensus
Each peer Pk stores chain Ck = (B1, ..., Bn)
Accept Ck iff ∀i : H(Bi) < T ∧ Bi.prev = H(Bi−1)
Connection to Thermoeconomics

Distributed consensus is an entropy-minimizing protocol. The honest chain represents the state with the most cumulative work (lowest entropy path through computation space). A malicious peer would need to expend more energy than all honest peers combined — a thermodynamic impossibility at scale, securing the network through physics rather than trust.

III · Transactions

Token Transfers

Replace free-text data with structured transactions. Each block records a set of value transfers between addresses. The hash commits to the exact transaction set — any alteration breaks the chain.

Definition — Transaction
txj = (fromj, toj, vj)
Bi.data = {tx1, ..., txk}
H(Bi) = SHA-256(i ∥ ηi ∥ serialize(txs) ∥ H(Bi−1))
Connection to Thermoeconomics

Transactions encode economic state transitions. Each transfer is a reallocation of value between agents. The hash function binds these state transitions to irreversible computation, creating a thermodynamic record of economic activity. The energy cost of mining secures not just data, but economic history itself.

III · Transactions

Coinbase Transaction

Every block begins with a special coinbase transaction — new tokens created from nothing, paid to the miner as a reward. This is the only mechanism for new currency to enter circulation, directly linking money creation to proof-of-work.

Definition — Coinbase
tx0 = (∅, miner, R), R = block reward
tx0 has no sender — tokens are created, not transferred
∀j > 0 : txj = (fromj, toj, vj) — standard transfers
Connection to Thermoeconomics

The coinbase transaction is where energy becomes money. The miner expends physical energy (electricity → heat) performing proof-of-work, and in return receives newly minted tokens. This is the thermodynamic anchor of the entire monetary system — currency is literally created through entropy production. Every token in circulation traces back to a coinbase transaction, and therefore to a specific quantity of dissipated energy.

VI · Algorithm Internals

SHA-256 Deep Dive

Step inside the SHA-256 algorithm. Watch message padding, schedule expansion, and all 64 compression rounds operate on your input in real time. Every hash you've seen in this toolkit was computed by exactly these operations.

FIPS 180-4 — Secure Hash Algorithm 256
SHA-256 : {0,1}* → {0,1}256
Steps: pad(m) → parse into 512-bit blocks → expand to 64 words → 64 compression rounds → digest
m Input Message
H(m) Final Digest
Step 1 — Pre-Processing (Padding)
Operation
m ∥ 1 ∥ 0k ∥ ⟨len⟩64
where k is smallest non-negative integer such that len + 1 + k ≡ 448 (mod 512)
■ message bytes ■ padding (0x80 + zeros) ■ length (64-bit big-endian)
Step 2 — Initial Hash Values H(0)
Definition
First 32 bits of the fractional parts of the square roots of the first 8 primes:
√2, √3, √5, √7, √11, √13, √17, √19
Step 3 — Message Schedule Wt
Expansion
Wt = Mt(i) for 0 ≤ t ≤ 15
Wt = σ1(Wt−2) + Wt−7 + σ0(Wt−15) + Wt−16 for 16 ≤ t ≤ 63
Step 4 — Compression Function (64 Rounds)
Round Function
T1 = h + Σ1(e) + Ch(e,f,g) + Kt + Wt
T2 = Σ0(a) + Maj(a,b,c)
Shift: h←g, g←f, f←e, e←d+T1, d←c, c←b, b←a, a←T1+T2
Round 0 / 63
Step 5 — Final Addition
Output
Hi(1) = Hi(0) + {a,b,c,d,e,f,g,h} after round 63
digest = H0 ∥ H1 ∥ ... ∥ H7
Reference — Bitwise Operations
Logical Functions
Ch(e,f,g)=(e AND f) XOR (NOT e AND g)"e chooses f or g per bit"
Maj(a,b,c)=(a AND b) XOR (a AND c) XOR (b AND c)"majority vote per bit"
Rotation & Shift Functions
Σ0(a)=ROTR2 XOR ROTR13 XOR ROTR22
Σ1(e)=ROTR6 XOR ROTR11 XOR ROTR25
σ0(x)=ROTR7 XOR ROTR18 XOR SHR3schedule expansion
σ1(x)=ROTR17 XOR ROTR19 XOR SHR10schedule expansion
Connection to Thermoeconomics

This is the engine of proof-of-work. Every mining trial executes these exact 64 rounds, each performing irreversible bit operations that dissipate energy per Landauer's principle. The avalanche effect (one input bit → ~50% output bits flip) is what makes brute-force the only viable strategy, ensuring that the energy cost of mining is real and cannot be shortcut. The thermodynamic cost of each hash is the physical foundation of blockchain security.
